OneSwipe ™

Enables linking offers to payment cards, and allows cardholders to redeem these offers directly at POS, creating a cash-back on their statement.

Cardholder shopping experience

How it works

Process:
  1. Issuer creates offer for targeted cardholders
  2. Cardholder on-boarding
    1. Cardholder in on-boarded at acquirer or VOP using card number
    2. Acquirer or VOP cardholder token is kept in Mobi724's database (we do not keep the card number because of PCI)
    3. Tokens are returned to issuing issuer for transactional reference
  3. Offer is created and assigned to cardholder
  4. Cardholder makes a purchase at participating merchant
  5. Acquirer or VOP sends transaction information to Mobi724â€™s transaction processing service
  6. Transaction processing service verifies that cardholder has at least one active offer for merchant. Transaction data is kept for data analysis
  7. Transaction processing
    1. If cardholder has at least one active offer for merchant, then transaction is sent for processing and result is returned after processing
    2. Post transaction to external message queue
    3. If cardholder does not have at least one active offer for merchant, then nothing is done
  8. Transaction is processed and if an offer can be applied, it is consumed and stored in Mobi724's database
  9. If an offer is applied, the communication service verifies if a confirmation message must be sent to cardholder
  10. Communication service triggers a communication event to an external CRM or marketing platform
  11. Communication service triggers a communication event using Mobi724's marketing platform
  12. A confirmation email or SMS is sent to the cardholder
  13. At end of each day, the statement credit service creates statement credit files
  14. Statement credit files are sent to the issuer or to Visa

Operations

Day to day operations consist of the Issuer maintaining members, merchants and offers in the Mobi724's platform.

A) To enroll members on the platform, the process is as follows:
  1. The Issuer must complete a .csv file with several fields for each member, some mandatory.
  2. The Issuer will deposit the file in the sFTP folder destined for that purpose.
  3. The Issuer will give notice to the Mobi724's system, through an e-mail, about the file deposited in the sFTP. Informing the name of this and the specific route.
  4. Mobi724 will execute the registration process between 24hs and 48hs after receipt of the notice by the Issuer.
  5. Depending on the number of records, the processing time of the platform may vary between 8 and 24 hours to be completed.
  6. Once finalized, a response file will be generated and deposited in a sFTP folder destined for this purpose. Mobi724 will notify the Issuer of the completion of the process.
  7. The Issuer must process this file and verify, both the records created correctly, and the records rejected and their causes in order to make the necessary corrections and adjustments, if desired, and then create a new file and repeat the process from the beginning until all the members on the platform have been enrolled.
